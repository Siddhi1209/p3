How to Explain SMART Portal
1. What is SMART Portal?
You can say:
SMART Portal is a monitoring dashboard used to check the health of banking payment systems, gateways, queues, servers, and transaction flow in real time.
It helps operations teams identify issues quickly and ensure banking services run smoothly.
You can also mention:
Centralized monitoring tool
Used for production support
Gives live system status
Helps detect failures, delays, queue buildup
2. What is Gateway?
Say this clearly:
Gateway is a communication bridge between two systems.
It receives transactions from one system and routes them to another system for processing.
Example:
Customer sends UPI payment → passes through UPI Gateway → sent to CBS/Core Banking for balance check and debit.
In your screen:
B24 Gateway
ATM Gateway
INB Gateway
MR Gateway
TF Gateway
These are different channels through which transactions enter.
Easy analogy:
Gateway is like a toll naka/checkpoint through which vehicles (transactions) pass.
3. What is Queue?
Say:
Queue is a waiting line of transactions/messages waiting to be processed.
When system is normal:
Queue count = Low / Zero
When issue occurs:
Queue count increases
Means messages are stuck or delayed
Example:
If 1000 UPI requests come but processing is slow, they wait in queue.
Analogy:
Queue is like people standing in line at a bank counter.
4. What is Queue Buildup?
Queue buildup means pending transactions are increasing faster than system can process.
Reasons:
Server slow
Application down
DB issue
Network issue
Dependency failure
Impact:
Delayed transactions
Failed payments
Customer complaints
5. How to Read This SMART Screen
Tell them:
Green Tick ✔
System healthy / running fine
Warning ⚠
Delay / partial issue / no data
Red / Cross
Down / critical issue
Queue Panel Right Side
Shows message queue counts for different queues.
If counts increase continuously → investigate immediately.
6. What Monitoring Team Does
Check gateways are green
Check queues are zero/stable
Check batch jobs running
Escalate if queue buildup
Restart services if SOP allows
Inform stakeholders
7. Sample Script for Your Session
This SMART portal is our real-time monitoring dashboard.
Different gateways represent banking transaction channels like ATM, internet banking, UPI etc.
These gateways transfer transactions to backend systems.
Queues are waiting lines where messages stay before processing.
If queue count increases, it means delay or issue in processing.
Our role is to monitor these indicators, identify abnormalities, troubleshoot, and escalate quickly.
8. If They Ask "Why Important?"
Because SBI handles huge transaction volumes. Even a small issue can impact many customers. Monitoring ensures smooth banking operations.
9. Pro Tip for Training
Do not go too technical first day. Explain flow:
Customer → Channel → Gateway → Queue → Application → Core Banking → Response
