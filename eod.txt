Training Script: Trickle Feed Process (Ready to Speak)
Good morning everyone. Today I will explain the Trickle Feed process used in banking operations.
Trickle Feed is a bulk file upload mechanism through which branches can process multiple customer transactions in one go instead of entering each transaction manually.
This is mainly used when a branch wants to upload many entries together such as:
Salary credits
Bulk debits
Clearing transactions
Financial and non-financial uploads
Large customer record updates
So in simple words, Trickle Feed converts a flat file into multiple transactions processed automatically by the core banking system.
Why Trickle Feed is Needed
Imagine if a branch has to credit salary for 500 employees manually. It would take hours and chances of human error would be high.
Instead, branch prepares one file containing:
Account numbers
Amounts
Customer details
Transaction type
That file is uploaded once, and system processes all records automatically.
So benefits are:
Saves time
Reduces manual effort
Faster processing
Less human error
Bulk transaction support
Overall Architecture Flow
The complete flow is:
1. Branch / Teller
Branch user creates and uploads file from teller menu.
2. Bhaga Server (Intermediate Server)
This acts as a secure middle layer between branch and production server.
Purpose:
Security
File routing
Prevent direct access to production
Manage uploads properly
3. FTP Upload Job
A background job continuously checks incoming files and transfers them to production servers.
4. Production / Core Server
Actual validation and transaction processing happens here.
5. Validation Stage
System checks:
File format correct or not
Duplicate file or not
Header/footer matched or not
Branch authorized or not
Record count correct or not
6. Processing Stage
Each record inside file is processed one by one.
Example:
Credit salary
Debit account
Post charges
Update customer data
7. Consolidated Report
After processing, system generates report containing:
Success records
Failed records
Rejected records
Error reasons
8. Report Sent Back to Branch
Final report is sent back to branch for confirmation.
Example Real Scenario
Suppose company has 1000 employees.
Branch uploads salary file.
System processes:
950 successful credits
30 invalid account failures
20 insufficient balance cases
Branch receives report instantly.

Common Failure Scenarios
1. File Not Reaching Production
Check:
FTP job running or not
Bhaga disk full or not
Network issue
2. Duplicate File Rejected
Same batch ID uploaded twice.
3. Validation Failed
Wrong format / record mismatch.
4. Queue Buildup
Too many files waiting.
5. Partial Processing
Some records success, some failed.
