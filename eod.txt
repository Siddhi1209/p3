Meaning of 755 and 777 Permissions
In UNIX/Linux, permissions are represented by three digits:
1st digit = Owner
2nd digit = Group
3rd digit = Others
Each digit is made from:
Read = 4
Write = 2
Execute = 1
So we add them.
755 Permission
Bash
chmod 755 file.sh
Breakdown:
7 = 4+2+1 = rwx → Owner has full access
5 = 4+1 = r-x → Group can read and execute
5 = 4+1 = r-x → Others can read and execute
So:
Bash
rwxr-xr-x
Meaning:
Owner can read, edit, run
Others can only read and run
Others cannot modify file
Used For:
Shell scripts
Application executables
Public readable files
Example:
Bash
runjob.sh
Owner can modify script, others can run it.
777 Permission
Bash
chmod 777 file.sh
Breakdown:
7 = rwx
7 = rwx
7 = rwx
So:
Bash
rwxrwxrwx
Meaning:
Everyone has full access:
Read
Write
Execute
Any user can:
open file
edit file
delete contents
run it
Risk:
Very unsafe in production.
Anyone can change critical files.

Banking Example
755
Batch script:
Bash
salary_run.sh
Others can execute but not modify.
777
If config file given 777:
Anyone can edit server config. Dangerous.
Interview Line
755 gives controlled shared access. 777 gives unrestricted access.
Best Practice
Use:
755 for scripts
644 for files
600 for passwords/private files
Avoid 777 unless temporary troubleshooting.
One-Line Closing
“755 is professional permission, 777 is risky permission.”
