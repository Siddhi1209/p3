
UNIX File System (Detailed & Easy Training Explanation)
Now I will explain the UNIX File System.
This is one of the most important basics because every application, script, log, job, and configuration file is stored somewhere in the file system.
If we understand file system well, we can troubleshoot faster.
1. What is File System?
A file system is the way operating system organizes files and folders on disk.
Just like in an office:
Cupboard contains drawers
Drawers contain files
Files contain documents
Similarly in UNIX:
Disk contains directories
Directories contain subdirectories
Directories contain files
So file system is the storage structure.
2. UNIX Uses Tree Structure
UNIX file system is like an upside-down tree.
Top level starts from:
Bash
/
This is called Root Directory.
All folders come under root.
Example:
Bash
/
/home
/etc
/var
/usr
/tmp
/opt
So everything begins from /
3. Difference Between Root Directory and Root User
Very common confusion.
Root Directory:
Bash
/
Top folder in file system.
Root User:
Admin user having highest privileges.
These are different things.
4. Paths in UNIX
Absolute Path
Starts from root /
Example:
Bash
/home/applprod/logs/app.log
Complete exact location.
Relative Path
Starts from current location.
Example:
Bash
logs/app.log
5. Important Standard Directories
Now explain each folder slowly.
/bin
Contains basic system commands.
Examples:
Bash
ls
cp
mv
cat
pwd
These command executables are stored here.
Think of /bin as toolbox.
/etc
Contains system configuration files.
Examples:
hostname
users
network settings
startup config
Think of /etc as control room settings.
Very important in server administration.
/home
Contains personal directories of users.
Example:
Bash
/home/user1
/home/applprod
/home/admin
Each user stores files here.
Think of it as employee cabins.
/tmp
Temporary storage.
Used for:
temp files
intermediate processing
extracts
scratch data
Files may get cleaned automatically.
Think of /tmp as temporary desk space.
/var
Variable data that changes regularly.
Contains:
logs
spool files
mail queues
runtime files
Important path:
Bash
/var/log
Used heavily for troubleshooting.
Think of /var as daily activity records.
/usr
Contains utilities, programs, libraries.
Usually large standard software area.
Think of it as software warehouse.
/opt
Optional third-party applications installed here.
Examples:
Oracle
Middleware
Banking products
Custom tools
Think of /opt as application zone.
/dev
Contains device files.
Examples:
disk
terminal
USB
printers
UNIX treats devices like files.
/proc
Contains process/system runtime information.
Useful for performance and troubleshooting.
6. Banking Application Example Structure
In production servers, you may see:
Bash
/finacle
/app
/interfaces
/logs
/scripts
/archive
/batch
Meaning:
interfaces = incoming files
scripts = shell jobs
logs = output/error logs
archive = old processed files
batch = scheduled jobs
7. Directories and Subdirectories
Example:
Bash
/home/applprod/logs/daily/error.log
Break it:
/ root
home main folder
applprod subfolder
logs subfolder
daily subfolder
error.log file
Like address hierarchy.
8. Special Symbols in File System
. Current Directory
Bash
./run.sh
Means run script in current folder.
.. Parent Directory
Bash
cd ..
Go one level up.
~ Home Directory
Bash
cd ~
Moves to user home.
9. Mount Points (Good Extra Topic)
Different disks may be attached under folders.
Example:
Bash
/data
/backup
/archive
These may actually be separate storage devices.
Use:
Bash
df -h
To check disk usage.
10. Why File System Important in Support
Because many issues are related to location/path/storage.
Example 1: File Not Found
Wrong path used.
Example 2: Job Failed
Log file in another folder.
Example 3: Disk Full
Filesystem 100%.
Example 4: Interface Pending
Input files stuck in inbound directory.
11. Real Commands Used with File System
Check Current Location
Bash
pwd
List Files
Bash
ls -ltr
Move Folder
Bash
cd /logs
Search File
Bash
find / -name abc.txt

13. Golden Rule
In UNIX:
Everything is stored as file or directory.
Even devices, logs, scripts, configs are handled like files.
